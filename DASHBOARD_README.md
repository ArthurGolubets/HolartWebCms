# Dashboard с метриками и виджетами

## Обзор

Реализована полнофункциональная система дашборда с настраиваемыми виджетами для отображения различных метрик системы.

## Возможности

### Базовые виджеты (всегда доступны):
- **Статистика пользователей** - Общее количество администраторов, активных и неактивных
- **Популярные страницы** - Топ страниц по посещениям (требует модуль Pages)
- **Последние действия** - История действий администраторов из логов

### Виджеты модуля Commerce:
- **Общая статистика заказов** - Количество всех заказов
- **Новые заказы** - Заказы со статусом "pending"
- **Выручка** - Общая сумма всех заказов
- **Ожидают оплаты** - Сумма заказов со статусом payment "pending"
- **Последние заказы** - Список из 5 последних заказов
- **Популярные промокоды** - Топ промокодов по использованию

### Виджеты модуля Callback:
- **Количество подписок** - Всего email подписок, активных и отписавшихся
- **График подписок** - Динамика подписок за последние 30 дней
- **Количество отзывов** - Всего отзывов, модерированных и ожидающих
- **Количество обращений** - Обращения от пользователей (всего, сегодня, за неделю)
- **Последние обращения** - Список из 5 последних обращений

## Возможности для супер-админа

Супер-администратор может:
1. **Настраивать виджеты** - Добавлять/удалять виджеты на дашборде
2. **Сбрасывать настройки** - Вернуть виджеты к настройкам по умолчанию
3. Виджеты автоматически подстраиваются под установленные модули

## Архитектура

### Backend

#### Модели:
- `TDashboardWidget` - Хранение настроек виджетов пользователя
  - Поля: admin_id, widget_type, position, settings, is_visible, width

#### Сервисы:
- `DashboardMetricsService` - Сбор всех метрик системы
  - Методы для каждого типа метрик (getUsersStats, getOrdersStats и т.д.)
  - Автоматическая проверка установленных модулей

#### Контроллеры:
- `DashboardMetricsController` - API для получения метрик
- `DashboardWidgetsController` - CRUD операции с виджетами

#### API Endpoints:
```
GET  /admin/api/dashboard/metrics - Получить все метрики
GET  /admin/api/dashboard/metrics/{type} - Получить конкретную метрику

GET  /admin/api/dashboard/widgets - Получить виджеты пользователя
GET  /admin/api/dashboard/widgets/available-types - Доступные типы виджетов
POST /admin/api/dashboard/widgets - Добавить виджет
PUT  /admin/api/dashboard/widgets/{id} - Обновить виджет
DELETE /admin/api/dashboard/widgets/{id} - Удалить виджет
POST /admin/api/dashboard/widgets/reorder - Изменить порядок виджетов
POST /admin/api/dashboard/widgets/reset - Сбросить к настройкам по умолчанию
```

### Frontend

#### Vue компоненты:
- `Dashboard.vue` - Главный компонент дашборда
- `dashboard/StatWidget.vue` - Виджет для отображения статистики
- `dashboard/ListWidget.vue` - Виджет для списков
- `dashboard/WidgetCard.vue` - Базовая карточка виджета

#### Особенности:
- Динамическая загрузка виджетов пользователя
- Автоматическая загрузка метрик
- Модальное окно для добавления виджетов (только для super_admin)
- Поддержка темной темы
- Адаптивный grid layout (12 колонок)

## Миграции

```sql
t_dashboard_widgets:
  - id
  - admin_id (foreign key to t_administrators)
  - widget_type (varchar 50)
  - position (integer)
  - settings (json)
  - is_visible (boolean)
  - width (integer 1-12)
  - timestamps
```

## Использование

### Добавление новых виджетов

1. Добавьте константу типа в `TDashboardWidget`:
```php
const TYPE_MY_WIDGET = 'my_widget';
```

2. Добавьте метод в `DashboardMetricsService`:
```php
public function getMyWidgetData(): array
{
    // Ваша логика
    return ['data' => ...];
}
```

3. Добавьте в `getMetricByType()`:
```php
'my_widget' => $this->getMyWidgetData(),
```

4. Обновите `Dashboard.vue` с рендерингом нового виджета:
```vue
<StatWidget
    v-else-if="widget.widget_type === 'my_widget'"
    title="Мой виджет"
    :value="metrics.my_widget?.data || 0"
    icon="chart"
    color="blue"
/>
```

## Установка

Миграция автоматически применится при запуске:
```bash
php artisan migrate
```

При первом входе пользователю автоматически создаются виджеты по умолчанию.

## Дополнительные метрики

Вы можете легко добавить свои метрики:

1. Создайте метод в `DashboardMetricsService`
2. Добавьте в `getAllMetrics()` и `getMetricByType()`
3. Создайте тип виджета в модели
4. Добавьте рендеринг во фронтенд

## Примечания

- Виджеты автоматически скрываются, если соответствующий модуль не установлен
- Каждый пользователь имеет свой набор виджетов
- Ширина виджетов настраивается от 1 до 12 колонок
- Порядок виджетов определяется полем `position`
